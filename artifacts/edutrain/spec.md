# EduTrain — 반복학습 & 적응형 출제 (MVP)

## 개요
학습자가 자기 자료를 넣으면 객관식·단답·서술형이 섞인 세트를 생성해 풀고, 문항마다 즉시 채점·해설을 받는다. 틀린 주제는 다음 세트에 우선 출제되고, 누적 점수·스트릭으로 반복 동기를 만든다. 모든 데이터는 이 브라우저에만 저장된다(나 혼자 MVP).

## 범위

### 포함
- 텍스트 붙여넣기 또는 `.txt`/`.md` 업로드로 학습자료 입력·저장
- 저장된 자료 목록에서 선택해 반복 학습
- 유형(객관식/단답/서술형)·난이도·문항 수를 골라 세트 생성
- 문항별 즉시 채점: 객관식·단답은 결정적, 서술형은 Gemini rubric 채점 + 피드백
- 세트 결과: 점수, 약점 주제 태그, 성과 기반 텍스트 추천
- 약점 주제 태그 기반 재출제
- 누적 점수 + 스트릭(연속 학습일 + 최고 연속 정답) + 지난 세션 점수 이력
- 브라우저 로컬 영속 저장

### 제외
- 로그인·서버 DB·다중 사용자 랭킹 — "나 혼자 MVP" 결정. 핵심 루프 검증 후 확장.
- PDF·문서 업로드 — 파싱은 별도 리스크 덩어리. `.txt`/`.md`만. (fast-follow)
- 니즈 입력 → 자료 자동 생성, 추천 → 자료 생성 — 별개 기능. 주어진 자료로 잘 도는 것부터 증명.
- 뱃지·리그·시즌 등 정교한 게임화 — 점수·스트릭이 "한 번 더"를 만드는지부터 확인.
- 기기 간 동기화 — 로컬 저장이라 해당 없음.

## 시나리오

### S1. 자료 입력 & 세트 생성
- **Given**: 앱 홈, 자료 입력 영역이 보이는 상태
- **When**: 자료를 입력/업로드하고 유형·난이도·문항 수를 고른 뒤 "세트 생성"을 누른다
- **Then**
  - [ ] S1-1 텍스트 자료 붙여넣기 + 유형[객관식] + 난이도[중] + 문항수[5] → "세트 생성" → 문항 5개짜리 세트가 만들어지고 첫 문항이 표시된다
  - [ ] S1-2 `.txt` 파일 업로드 → 파일 내용이 자료 입력 영역에 반영되고, 그 자료로 세트를 생성할 수 있다
  - [ ] S1-3 자료가 비어 있는 상태로 "세트 생성" → "학습자료를 입력하세요"(제안 기본값)가 표시되고 세트가 생성되지 않는다
  - [ ] S1-4 유형을 객관식+서술형 둘 다 선택 → 생성된 세트에 두 유형의 문항이 함께 포함된다
  - [ ] S1-5 생성된 각 문항에는 자료에서 도출된 주제 태그가 하나 이상 붙어 있다
  - [ ] S1-6 세트 생성(Gemini 호출) 실패/한도초과 → 에러 안내와 "다시 시도" 버튼이 표시되고, 앱은 홈 상태를 유지한다

### S2. 문항 풀이 & 즉시 채점 — 객관식·단답
- **Given**: 세트의 객관식 또는 단답 문항이 표시된 상태
- **When**: 답을 고르거나 입력하고 "제출"을 누른다
- **Then**
  - [ ] S2-1 객관식 정답 보기 선택 → 제출 → "정답"(제안 기본값) 표시 + 해설 + "다음 문항" 이동 수단이 나타난다
  - [ ] S2-2 객관식 오답 보기 선택 → 제출 → "오답"(제안 기본값) + 정답 보기 표시 + 해설이 나타난다
  - [x] S2-3 단답형에 정답 문자열 입력(대소문자·앞뒤 공백 차이 허용) → 제출 → 정답으로 처리된다
  - [ ] S2-4 제출 후에는 같은 문항의 답을 다시 바꿀 수 없다(채점 확정)

### S3. 문항 풀이 & 즉시 채점 — 서술형 (Gemini rubric)
- **Given**: 세트의 서술형 문항이 표시되고 답안이 작성된 상태
- **When**: "제출"을 누른다
- **Then**
  - [ ] S3-1 답안 제출 → 채점 중 로딩 표시 후, 부분점수(제안 기본값: 0~100%)와 피드백 텍스트가 표시된다
  - [ ] S3-2 빈 답안으로 제출 → "답안을 입력하세요"(제안 기본값)가 표시되고 채점이 호출되지 않는다
  - [ ] S3-3 서술형 채점(Gemini) 실패/한도초과 → 에러 안내 + "다시 채점" 버튼이 표시되고, 이미 푼 문항 결과와 세션 진행은 유지된다

### S4. 세트 결과 & 약점 & 추천
- **Given**: 세트의 모든 문항을 풀고 채점이 끝난 상태
- **When**: 마지막 문항 제출 후 결과 화면으로 진입한다
- **Then**
  - [ ] S4-1 → 세트 점수(정답 수/전체 또는 백분율, 제안 기본값)가 표시된다
  - [ ] S4-2 → 틀린 문항의 주제 태그가 "약점 주제"로 묶여 표시된다
  - [ ] S4-3 세트 점수가 임계값 이상(제안 기본값: 80%) → "다음으로 이 주제를 공부해보세요" 형태의 텍스트 추천이 표시된다
  - [ ] S4-4 → "약점만 다시 풀기" 수단을 누르면 약점 주제 태그 위주의 새 세트가 생성된다

### S5. 반복 학습 & 진척 (점수·스트릭·이력)
- **Given**: 이전에 하나 이상의 세트를 완료한 기록이 있는 상태
- **When**: 홈에 진입한다
- **Then**
  - [ ] S5-1 → 누적 점수가 표시된다
  - [ ] S5-2 → 스트릭(연속 학습일과 최고 연속 정답)이 둘 다 표시된다
  - [ ] S5-3 → 지난 세션들의 점수 목록(추이)이 표시된다
  - [ ] S5-4 저장된 자료 목록에서 자료를 선택 → 그 자료로 새 세트를 생성할 수 있다
  - [x] S5-5 오늘 첫 세트를 완료 → 연속 학습일이 1 증가한다(같은 날 추가 완료는 증가시키지 않는다)

### S6. 약점 태그 기반 재출제
- **Given**: 특정 주제 태그에서 오답 기록이 있는 상태
- **When**: 같은 자료로 새 세트를 생성한다(또는 "약점만 다시 풀기")
- **Then**
  - [ ] S6-1 → 새 세트에는 약점 태그에 속한 문항이 그렇지 않은 태그보다 더 많이 출제된다
  - [x] S6-2 약점 태그의 문항을 이후 세트에서 맞힘 → 그 태그가 약점 목록에서 빠지거나 완화된다

## 불변 규칙

- [ ] INV-1 브라우저 네트워크 응답과 클라이언트 번들 어디에서도 Gemini API 키가 노출되지 않는다 (보안)
- [ ] INV-2 새로고침하거나 다시 방문해도 누적 점수·스트릭·저장된 자료·세션 이력이 유지된다 (로컬 영속성)
- [x] INV-3 화면에 표시되는 누적 점수·스트릭은 완료된 세션 기록과 항상 일치한다 (데이터 일관성)

## End-to-end 검증

개발 서버(`bun run dev`)를 띄우고 `http://localhost:3000` 진입 후:

1. 텍스트 자료를 붙여넣고 유형[객관식+서술형]·난이도[중]·문항수[5]로 세트 생성 → 첫 문항 표시, 각 문항에 주제 태그 (S1)
2. 객관식 정답/오답을 각각 제출 → 즉시 "정답/오답"·정답·해설 표시, 제출 후 변경 불가 (S2)
3. 서술형 답안 제출 → 부분점수·피드백 표시; 빈 답안·채점 실패 처리 확인 (S3)
4. 마지막 문항까지 풀어 결과 진입 → 세트 점수·약점 주제·(고득점 시)텍스트 추천·"약점만 다시 풀기" (S4)
5. 홈 재진입 → 누적 점수·스트릭(둘 다)·지난 세션 이력·저장 자료 목록 (S5)
6. 저장 자료로 새 세트 생성 → 약점 태그 문항이 더 많이 출제됨 확인 (S6)
7. 페이지 새로고침 → 데이터 유지(INV-2), 네트워크 탭에서 API 키 미노출(INV-1), 표시 점수와 기록 일치(INV-3)

## 의존성

- `GEMINI_API_KEY` 환경변수 (설정 완료)
- `@google/genai` 패키지 (설치 완료)
- 인터넷 연결 (Gemini 호출)

## 미결정 항목

- 서술형 채점 rubric을 Gemini가 문항마다 자동 생성할지, 자료 기반 고정 rubric을 뽑을지 — 채점 신뢰성 spike 결과에 따라 plan/구현 단계에서 결정 (idea.md의 가장 위험한 가정과 직결).
- S3-1의 서술형 점수 표현(0~100% vs 상/중/하)과 S4-3의 추천 임계값(80%)은 제안 기본값이며, 초기 사용 후 조정 가능.
